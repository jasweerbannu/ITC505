LAB 3: SERVING FONTS

JOKERMAN FONT

I am using the @font-face rule, which is a CSS rule allowing developers to define custom fonts for their websites. It ensures consistent typography across all devices and different browsers.

Initially, we need to check if the font is available in various formats like 'WOFF2', 'WOFF', 'TTF', etc., to ensure compatibility across different browsers. Older browsers generally support 'TTF' or 'OTF', while modern browsers support 'WOFF2' and 'WOFF'.

Then, in our CSS file, we define the @font-face rule to specify the font-family name ('Jokerman') and the source URLs for the font files. This allows the browser to download the font if it's not already available locally.

Once we've defined the @font-face, we can use our custom font in our CSS using the font-family property. If our custom font isn't loaded for any reason, we provide a fallback mechanism to ensure another font is rendered. Here I added the "sans-serif" as fallback mechanism font.
